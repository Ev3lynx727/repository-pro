## Topic Naming Rules

1. **Use lowercase** - GitHub normalizes all topics to lowercase
2. **Use hyphens** - Separate words with hyphens: `machine-learning`
3. **No special characters** - Only letters, numbers, hyphens
4. **Max 35 characters** - Per topic limit
5. **Max 20 topics** - GitHub repository limit

## Bad vs Good Topics

| Bad | Good |
|-----|------|
| ReactJS | react |
| Python_Library | python |
| My-Awesome-Tool | awesome-tool |
| API_REST | rest-api |

## SEO-Friendly Descriptions

### Template: "[Action] [what] with [technology/benefit]"

| Project | Good Description |
|---------|-----------------|
| CLI tool | "CLI tool to resize images with batch processing" |
| Library | "TypeScript library for building CLI applications" |
| Web app | "Real-time chat application built with React and Socket.io" |
| API | "REST API for managing personal finance budgets" |
| Bot | "Discord bot for playing music in voice channels" |

### Description Tips

- Keep under 160 characters
- Include primary action: "A", "An", "Build", "Create", "Simple"
- Mention the technology stack
- Highlight the main benefit
